Proyecto El Niño

Plataforma gamer personalizada para convertir un cumpleaños en una experiencia digital moderna, emocional y escalable, mezclando misiones, recuerdos, videos, música, sorpresas y narrativa familiar.

Ver misiones Explorar recuerdos

Objetivo principal

Una celebración convertida en aventura

El objetivo de la plataforma es transformar un cumpleaños en una experiencia interactiva con estética gamer, donde cada página funciona como una zona desbloqueable: inicio, misiones, videos, recuerdos, regalo digital, rincón gamer y sorpresa final.

La web no se plantea solo como una felicitación, sino como una pequeña campaña digital pensada para que El Niño se sienta protagonista de su propio universo.

Visión

De web especial a plataforma ampliable

El proyecto empieza como una web estática rápida, visual y fácil de mantener, pero está preparado para crecer hacia una plataforma con datos dinámicos, panel de administración, contenido familiar gestionable y nuevas experiencias desbloqueables.

La idea es que pueda evolucionar con el tiempo: añadir nuevas temporadas, nuevas misiones, más recuerdos, rankings, minijuegos o integraciones externas.

Experiencia de usuario

Diseñada como si fuera una partida

La navegación está pensada para que el usuario avance como en un videojuego: entra, explora, desbloquea contenido, descubre recuerdos y llega a una recompensa. Los botones, tarjetas, videos de fondo, música y textos usan lenguaje de misión para reforzar esa sensación.

El tono mezcla emoción familiar con estética digital: neones, paneles, insignias, progreso, botín, comunidad y final épico.

Arquitectura

Escalabilidad

El proyecto está separado por páginas, estilos, datos, servicios y configuración para poder crecer sin desorden.

Fase 1

Base estática con HTML, CSS y JavaScript modular para tener una plataforma rápida, ligera y fácil de publicar.

Fase 2

Contenido más dinámico mediante listas de videos, datos centralizados, secciones reutilizables y mejoras de interacción.

Fase 3

Integración con APIs externas como YouTube, almacenamiento de imágenes y formularios para recibir mensajes familiares.

Fase 4

Migración a framework y backend si el proyecto necesita usuarios, login, base de datos, panel privado o estadísticas reales.

Identidad visual

Estilo gamer familiar

La plataforma usa una estética de interfaz futurista con colores intensos, tipografía de videojuego, tarjetas oscuras, sombras, fondos multimedia y llamadas a la acción con tono de campaña.

El resultado busca ser divertido y moderno sin perder el sentido principal: celebrar a una persona concreta con cariño, recuerdos y detalles personalizados.

Contenido administrable

Preparada para cambiar videos y textos

La estructura permite sustituir videos, mensajes, títulos y secciones sin rehacer toda la página. Esto facilita ajustar la plataforma hoja por hoja, ampliar contenido y personalizar cada zona según la celebración.

Integraciones

YouTube Data API v3 - Sistema preparado

La plataforma esta preparada para conectarse a la YouTube Data API v3 de Google. Actualmente funciona con listas de videos estaticas (sin consumir cuota), pero el sistema esta disenado para que cualquier usuario pueda activar la API real y cargar playlists dinamicas.

Que ofrece la YouTube Data API v3?

  • Acceso programatico a videos, playlists, canales y estadisticas de YouTube.
  • Busqueda de contenido por palabras clave, canal o ID de lista.
  • Obtencion de miniaturas, titulos, descripciones y metadatos en tiempo real.
  • Compatible con JavaScript mediante solicitudes REST (fetch / axios).

Como obtener tu API Key

  1. Ve a Google Cloud Console.
  2. Crea un proyecto nuevo o selecciona uno existente.
  3. Activa la API YouTube Data API v3 en la biblioteca de APIs.
  4. Ve a Credenciales y genera una Clave de API (restringe por HTTP referer para seguridad).
  5. Copia tu clave y configurala en el archivo de variables del proyecto.

Cuotas y limites

Google asigna 10.000 unidades de cuota al dia por defecto (gratuito). Cada operacion consume unidades: una busqueda cuesta 100 unidades, obtener detalles de una lista 1 unidad por elemento, y la lista de videos de un canal 1-3 unidades. Con uso moderado, 10.000 unidades cubren miles de visitas diarias.

Estado de la plataforma

  • Fase actual: listas estaticas definidas en src/js/data/youtube-playlist.js. Sin necesidad de API key ni cuota.
  • Preparacion: el modulo src/js/services/youtube-service.js ya esta estructurado para aceptar peticiones a https://www.googleapis.com/youtube/v3/.
  • Proximo paso: sustituir las listas estaticas por llamadas a la API usando la key del usuario, o crear un archivo .env / config.js donde cada usuario introduzca su propia clave.

El sistema esta listo. Solo necesitas tu API Key para pasar de listas estaticas a contenido dinamico de YouTube.
